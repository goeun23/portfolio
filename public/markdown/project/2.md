#### POSITION
- __Multi-SPA 아키텍처 설계__: 기능 단위 SPA 분리, 공통 Core(인증/상태/컴포넌트)로 재사용 구조 확립
- __Vite 단일 설정 + 선택 빌드__:  vite.config와 git diff 기반 빌드 스크립트로 구성하여 변경된 프로젝트만 빌드
- __API 계약/응답 표준화__: API 인터셉터로 응답·에러·인증 만료 처리 일원화
- __인증/라우팅 안정화__: 로그인 Mixin + Router Guard(만료·재인증·공통 Fallback)
- __운영/모니터링 토대__: 라우팅 전 헬스체크 호출, 에러·API 예외 로깅 훅 구성
- __배포·세션 전략__: CloudFront Prefix로 SPA 분리 서빙, 동일 도메인 세션 공유

#### 문제-조치-결과
__서버 의존적인 프로젝트 독립적 분리__
- 기능별 Multi-SPA 분리, 단일 Vite 설정 + 선택 빌드
- 빌드/배포 리드타임 −80%, 변경 없는 SPA 스킵 배포로 운영 부담 감소

__API 응답·에러 처리 불일치로 MTTR 증가__
- 응답 포맷 표준화 + 인터셉터로 인증만료/에러코드 공통 처리
- 에러 트래킹 시간 단축(−60%), 프런트/서버 에러 구체화

__인증/라우팅 연동(새로고침·세션 만료·다중 SPA 이동)__
- Mixin + Guard로 진입 검증·만료 처리·공통 Fallback
- 세션 연속성 ↑, 재로그인/오동작 CS 해소

__모니터링 블라인드__
- 헬스체크 선 호출 + 프런트 에러/거부 태깅 로깅
- MTTD −75%, 장애 원인 파악 속도 향상

__여러 SPA의 로그인/세션 일관성__
- 동일 도메인 CloudFront Prefix 서빙으로 쿠키/세션 공유
- SSO 경험 유지, 교차 이동 UX 개선

#### 성과
- 선택 빌드/부분 배포로 출시 리드타임 −65%, 운영/배포 리소스 −90%
- 인증/라우팅 안정화로 에러성 재방문/CS −88%
- 구조 표준화 후 파생 서비스 4건 안정 운영
