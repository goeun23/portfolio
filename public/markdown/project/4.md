
__렌더링 아키텍처 고도화 (Streaming SSR)__
- 문제 상황: 기존 CSR 방식의 Waterfall 현상으로 인해 초기 콘텐츠 노출까지 **1.5초 이상 지연 현상 발생**
- **해결 전략**:
  - **Streaming SSR 도입**: `Suspense` 경계를 활용해 UI를 청크(Chunk) 단위로 쪼개어, 준비된 UI(헤더, 사이드바 등)부터 즉시 브라우저로 전송.
  - **병렬 데이터 페칭(Parallel Fetching)**: 사용자 프로필, 캐릭터, 습관, 투두 데이터를 서버에 병렬로 요청하여 로딩 시간을 단축.

__UI 사용성 개선을 위한 Optimistic Update 적용__

- **문제 상황**: 잦은 상호작용 발생시 서버 응답을 기다리는 동안 UI 이질감 발생.
- **해결 전략**: Optimistic UI 패턴을 적용하여 사용자의 액션 즉시 UI 상태를 '성공'으로 가정하여 미리 업데이트, 실패 시 원래 상태로 자동 롤백되는 안정적인 에러 핸들링 로직 구현
- **결과**: 사용자 반응 속도(Interaction Latency)를 **Almost 0ms**로 단축하여 네이티브 앱과 유사한 경험 제공

__Server Actions 기반의 보안 및 아키텍처 개선__
- **구현 내용**: 클라이언트 컴포넌트에 노출되어 있던 API 호출 및 상태 변경 로직을 **Server Actions**로 이관.
- **결과**: 데이터베이스 직접 통신 로직을 서버 내부로 숨겨 보안성을 강화하고, 클라이언트 번들 사이즈를 최적화.

#### 성과
- FCP 1.5 -> 0.3Sec 단축 
- TTFB 10.0 -> 3.0-5.0 ms 개선 
- LCP 2.0 -> 0.8sec 단축
